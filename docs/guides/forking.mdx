# Forking

Codeplane supports fork-based collaboration, but this repository currently prioritizes shared-repo bookmark stacks over a dedicated fork workflow.

Until the fork flow is documented in full:

- Use repository access controls to collaborate directly when possible.
- Prefer jj bookmark stacks and landing requests for reviewable work.
- Mirror to GitHub only when you need downstream GitHub-specific tooling.
