# GitHub CLI to Codeplane CLI Cheat Sheet

If you are transitioning from GitHub to Codeplane, the `codeplane` CLI provides a familiarly designed interface tailored specifically for Jujutsu (`jj`) workflows and agentic teams.

Here is a quick translation guide from the GitHub CLI (`gh`) to the Codeplane CLI (`codeplane`).

## Authentication & Setup

| Action | GitHub CLI (`gh`) | Codeplane CLI (`codeplane`) |
| ------ | ----------------- | ------------------- |
| Login | `gh auth login` | `codeplane auth login` |
| Logout | `gh auth logout` | `codeplane auth logout` |
| Check auth status | `gh auth status` | `codeplane auth status` |

## Repository Management

| Action | GitHub CLI (`gh`) | Codeplane CLI (`codeplane`) |
| ------ | ----------------- | ------------------- |
| Create a repo | `gh repo create` | `codeplane repo create <name>` |
| Clone a repo | `gh repo clone <repo>` | `codeplane repo clone <repo>` |
| List repos | `gh repo list` | `codeplane repo list` |
| View repo details | `gh repo view` | `codeplane repo view` |

## Pull Requests / Landing Requests

Codeplane uses **Landing Requests (LRs)** instead of Pull Requests, built native for `jj` stacked changes.

| Action | GitHub CLI (`gh`) | Codeplane CLI (`codeplane`) |
| ------ | ----------------- | ------------------- |
| Create a PR / LR | `gh pr create` | `codeplane land create` |
| List open PRs / LRs | `gh pr list` | `codeplane land list` |
| View PR / LR | `gh pr view <num>` | `codeplane land view <num>` |
| Review PR / LR | `gh pr review <num>` | `codeplane land review <num>` |
| Land PR / LR | `gh pr merge <num>` | `codeplane land land <num>` |
| View CI checks | `gh pr checks` | `codeplane land checks <num>` |

## Issue Tracking

| Action | GitHub CLI (`gh`) | Codeplane CLI (`codeplane`) |
| ------ | ----------------- | ------------------- |
| Create an issue | `gh issue create` | `codeplane issue create` |
| List issues | `gh issue list` | `codeplane issue list` |
| View an issue | `gh issue view <num>` | `codeplane issue view <num>` |
| Close an issue | `gh issue close <num>` | `codeplane issue close <num>` |

## Unique Codeplane Commands

Because Codeplane is built for agentic workflows and `jj` version control, the CLI includes features that do not have a direct GitHub equivalent:

- **AI Agents**: Use `agent()` steps in TypeScript workflows to automate tasks with Claude-powered AI agents.
- **Workflows**: Trigger CI/CD pipelines natively with `codeplane workflow run <id>` and `codeplane run watch <run-id>`.
- **Local Changes**: Operate on local `jj` history with `codeplane change list` and `codeplane bookmark list`.
