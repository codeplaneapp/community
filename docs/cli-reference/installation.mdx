# Installing the Codeplane CLI

The Codeplane CLI (`codeplane`) is a Bun-compiled binary (TypeScript) that operates in two modes:
- **Local mode**: Uses the `jj` CLI directly for repository operations
- **Remote mode**: Communicates with the Codeplane API for cloud operations

## Prerequisites

The Codeplane CLI requires **Jujutsu (jj)** to be installed and available on your `PATH`. 

- **macOS**: `brew install jj`
- **Linux/Other**: `cargo install jj-cli` or see the [jj installation guide](https://jj-vcs.github.io/jj/latest/install-and-setup/).

## macOS and Linux

The Codeplane CLI provides a unified installation script that automatically detects your operating system and architecture (x86_64 or arm64).

```bash
curl -fsSL https://get.codeplane.app/cli | bash
```



## Verify Installation

```bash
codeplane --version
codeplane auth status
```

## Configuration

The CLI reads configuration from:
1. `CODEPLANE_TOKEN` environment variable (for authentication)
2. `~/.config/codeplane/config.toon` (for defaults)
3. `.codeplane/config.toon` (repository-level settings)

## Shell Completion

```bash
# bash
codeplane completion bash >> ~/.bashrc

# zsh
codeplane completion zsh >> ~/.zshrc

# fish
codeplane completion fish > ~/.config/fish/completions/codeplane.fish
```
