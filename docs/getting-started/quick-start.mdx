# Quick Start

Get up and running with Codeplane - the jj-native code hosting platform with integrated collaborative agent sandboxes in the cloud.

## Prerequisites

- [jj (Jujutsu)](https://github.com/jj-vcs/jj) installed
- The Codeplane CLI (`codeplane`) installed - see [Installation](/cli-reference/installation)
- A Codeplane account - currently available via closed alpha invite

## 1. Authenticate

```bash
codeplane auth login
```

This opens your browser for GitHub OAuth authentication. Once authorized, your credentials are stored securely in your OS keychain.

Verify you're logged in:

```bash
codeplane auth status
```

For CI environments where a browser isn't available, use a personal access token:

```bash
export CODEPLANE_TOKEN=codeplane_your_token_here
```

## 2. Create a Repository

```bash
codeplane repo create my-project --description "My first Codeplane project"
```

By default, repositories are public. To create a private repo:

```bash
codeplane repo create my-private-project --private
```

## 3. Clone and Work

```bash
codeplane repo clone username/my-project
cd my-project
```

Edit files, then describe your change (jj doesn't use `git add` or `git commit`):

```bash
# Edit a file
echo "# My Project" > README.md

# Describe your change
jj describe -m "Add README"

# Push to Codeplane
jj git push
```

## 4. Open a Landing Request

Landing Requests (LRs) are Codeplane's replacement for Pull Requests, built natively for jj's stacked changes model.

```bash
codeplane land create --title "Add README" --target main
```

To land a whole stack of changes:

```bash
codeplane land create --title "Feature: user auth" --target main --stack
```

## 5. Review and Merge

View your landing request:

```bash
codeplane land view 1
```

Check CI status:

```bash
codeplane land checks 1
```

Land when ready:

```bash
codeplane land land 1
```
