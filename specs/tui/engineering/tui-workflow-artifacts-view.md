# Engineering Specification: TUI_WORKFLOW_ARTIFACTS_VIEW

## Overview
This specification outlines the technical implementation for the Workflow Artifacts View in the Codeplane TUI (`TUI_WORKFLOW_ARTIFACTS_VIEW`). The screen displays all artifacts generated by a specific workflow run, providing terminal-native capabilities to browse, inspect, download, and delete artifacts. It features real-time client-side sorting and filtering, responsive column adjustment, and integrated overlay modals for detail inspection and deletion confirmation.

## Architecture & State Management

### Data Flow
- **Hooks**: `useWorkflowRunArtifacts(owner, repo, runId)` to fetch the complete list. `useDeleteWorkflowArtifact(owner, repo, runId, artifactName)` for deletion.
- **Client-side Processing**: Artifact lists are typically small (<100 items), with a hard cap at 200 items. Sorting and filtering occur synchronously in memory.
- **Action Delegation**: Downloading is delegated to the CLI via a spawned Node.js `child_process` invoking `codeplane artifact download` with the current user's token context.

### Local State
- `filterStatus`: `'all' | 'ready' | 'pending' | 'expired'`
- `sortOrder`: Enum of `created_desc`, `created_asc`, `name_asc`, `name_desc`, `size_desc`, `size_asc`
- `searchText`: String query managed via the `/` keybinding and standard input capturing.
- `focusedArtifactId`: Tracks the currently highlighted row in the list.
- `overlayState`: `{ type: 'none' | 'detail' | 'delete', artifactId?: string }`

## Implementation Plan

### 1. Scaffold UI Components (`apps/tui/src/screens/Workflows/components/`)
1.  **`ArtifactRow.tsx`**: 
    - Build a stateless flexbox row component receiving the artifact object, `isFocused`, and `breakpoint`.
    - Implement conditional column rendering based on the `breakpoint`.
    - Map statuses to icons: `ready` → ● (green), `pending` → ◎ (yellow), `expired` → ○ (gray).
    - Build text truncation helpers that respect ANSI boundaries and grapheme clusters.
2.  **`ArtifactDetailOverlay.tsx`**: 
    - Build a modal using `ModalSystem` (60% × 50% dimensions, responsive min constraints).
    - Display all metadata in a `<scrollbox>`: name, type, exact/formatted size, timestamps, GCS path.
    - Add `D` (Download) and `x` (Delete) quick actions directly in the modal.
3.  **`ArtifactDeleteOverlay.tsx`**: 
    - Build a confirmation modal (40% × 25%, error border).
    - Handle the `isSubmitting` state to show a spinner and block multiple submissions.

### 2. Implement the Main View (`apps/tui/src/screens/Workflows/WorkflowArtifactsView.tsx`)
1.  **Context & Setup**: 
    - Retrieve `owner`, `repo`, `runId` from router params (`useNavigation`).
    - Fetch data using `useWorkflowRunArtifacts`.
2.  **Layout Structure**: 
    - **Title Row**: Render "Artifacts (N)" and compute `totalSizeBytes` format using a new `formatBytes` utility.
    - **Toolbar**: Render filter tags, sort mode, and the search input container.
    - **Header Row**: Render column titles, hiding fields based on `useLayout().breakpoint`.
    - **List Area**: Mount the generic `ScrollableList` component.
3.  **Client-Side Operations**: 
    - Apply `.filter()` for status and case-insensitive search text.
    - Apply `.sort()` based on the active `sortOrder` state.
    - Enforce the 200-item cap slice and render a warning in the list footer if exceeded.

### 3. Implement Keybindings & Interactions
Use the `useScreenKeybindings` hook to register Priority 4 handlers:
- `f`: `setFilterStatus(nextStatus)`
- `s`: `setSortOrder(nextSort)`
- `/`: Focus search `<input>`.
- `Enter`: Set `overlayState` to `{ type: 'detail', artifactId: focusedId }`.
- `D` (Shift+D): Dispatch `initiateDownload(focusedId)`.
- `x`: Set `overlayState` to `{ type: 'delete', artifactId: focusedId }`.
- `q` / `Esc`: Handle modal dismissal or screen `pop()` via priority logic.

### 4. Implement CLI Download Delegation
1.  **Validation**: Intercept if the artifact is `pending` or `expired`. Display a temporary status bar message and abort.
2.  **Execution**: 
    - Use Node.js `child_process.spawn`.
    - Command: `codeplane artifact download <runId> <name> --repo <owner>/<repo> --output <name>`.
    - Inherit the CLI's `CODEPLANE_TOKEN` via `process.env`.
3.  **UX**: Emit status bar updates: "Downloading...", and "Download completed" upon process exit code 0. Handle process failure gracefully.

### 5. Implement Deletion Action
1.  On `Enter` in the `ArtifactDeleteOverlay`, call `deleteArtifact(name)`.
2.  Leverage the optimistic update pattern in the mutation hook to instantly remove the row.
3.  On success, push a "Artifact deleted" message to the status bar and close the modal.
4.  On failure (e.g., 403, 409), revert the optimistic update and display the error.

### Edge Case Tests (14 tests)
- `EDGE-ART-001` - `EDGE-ART-004`: Environment edge cases (no token, extreme text strings, unicode/emoji processing, 1-item lists).
- `EDGE-ART-005` - `EDGE-ART-008`: High-stress rendering (concurrent resizes, disjointed searches, 60+ char content types).
- `EDGE-ART-009` - `EDGE-ART-014`: Action spamming (rapid `x`/`D` requests), auto-empty state transitions, zero-byte payloads, and race conditions (delete + download simultaneously).
