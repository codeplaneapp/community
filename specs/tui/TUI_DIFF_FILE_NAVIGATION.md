# TUI_DIFF_FILE_NAVIGATION

Specification for TUI_DIFF_FILE_NAVIGATION.

## High-Level User POV

File navigation within the diff viewer lets developers move between changed files using `]`/`[` keybindings for sequential next/previous navigation (with cyclic wrapping) and `Enter` in the file tree sidebar for targeted jumps. Both methods produce a coordinated state update: the main content scrolls to the target file's header, the sidebar highlights the active file, and the status bar shows "File N of M". Navigation works regardless of focus zone, when sidebar is hidden, and at all terminal sizes. Single-file diffs treat navigation keys as no-ops. The feature preserves hunk collapse state, whitespace toggle, and view mode across navigations. It is entirely client-side — no API calls are made during navigation.

## Acceptance Criteria

19 definition-of-done items: `]` advances focusedFileIndex (wraps last→first), `[` retreats (wraps first→last), navigation scrolls main scrollbox to target file header, updates sidebar highlight with reverse-video, updates status bar "File N of M", works from both content and tree focus zones, works with sidebar hidden, `Enter` in tree jumps to focused file and updates index/status bar, navigation preserves hunk state/whitespace/view mode, single-file diff makes `]`/`[` no-ops showing "File 1 of 1", sidebar scrollbox auto-scrolls to keep focused entry visible. 5 keyboard interactions (]/[/Enter/j-k/Tab). 8 boundary constraints: left-padded N, max 500 files, min 2 for navigation, file path truncation with `…/` prefix, status bar max 16 chars, modular arithmetic bounds, scroll offset calculation, K/M stat abbreviation. 16 edge cases covering empty diff, rapid presses, single-file, resize during navigation, binary files, loading/error states, inline comments preservation, renamed files, and whitespace toggle reducing file count.

## Design

Operates within the existing diff screen layout — no new components. Coordinates three visual zones: file tree sidebar (inverse styling on focused entry), main content scrollbox (scrollTo on file header ref), and status bar ("File N of M" indicator). Component structure uses `<box inverse={fileIndex === focusedFileIndex}>` for sidebar entries, `<scrollbox ref={mainScrollRef}>` with file header refs for scroll targeting, and `<text>` for status bar indicator. Keybindings registered via `useKeyboard()`: `]`/`[` active in any focus zone, `Enter` only when tree focused, `Tab` to switch focus. State: `focusedFileIndex` (bounded by modular arithmetic), no changes to `hunkStates`, `viewMode`, `whitespaceVisible`, or `focusZone`. Scroll mechanics: calculate cumulative heights of preceding file sections, scrollTo target offset. Responsive: sidebar hidden at 80×24 (navigation still works via status bar only), full three-zone at 120×40+, file paths truncated from left with `…/` at narrow widths. Data hooks: `useChangeDiff`/`useLandingDiff` for FileDiffItem[], `useTerminalDimensions` for sidebar visibility.

## Permissions & Security

File navigation is purely client-side on already-loaded data — no additional API calls or authorization checks. Repository read access is verified at diff load time; if the diff loaded, navigation works unconditionally. No tokens transmitted during navigation. Token expiry doesn't affect navigation (only subsequent operations like comments). No rate limit consumption. Input sanitization: `]`/`[` are single-char keybindings with no payload, focusedFileIndex bounded by modular arithmetic, file paths from already-loaded API response.

## Telemetry & Product Analytics

4 business events: `tui.diff.file_navigated` (direction, from/to index, total files, wrapped, focus zone, sidebar visible), `tui.diff.file_tree_selected` (from/to index, same_file flag), `tui.diff.file_nav_noop` (single-file attempts), `tui.diff.file_nav_pattern` (summary on screen close — unique files visited, sequential/tree nav counts, wrap count, max consecutive same direction). Common properties: session_id, terminal dimensions, timestamp, user_id, diff_source, repo. 7 success indicators with targets: file nav adoption >60% of multi-file sessions, tree selection >30%, files visited >50%, wrap rate <40%, file nav 15-35% of keystrokes, mixed navigation >15%, noop rate tracked for UX hints.

## Observability

6 log events at debug level: file.navigated, file.tree_selected, file.scroll_target (offset/viewport), file.sidebar_scroll, file.noop (with reason: single_file/loading/error). 1 info-level nav_summary on screen unmount. Logs to stderr, level controlled by CODEPLANE_LOG_LEVEL (default warn). 6 error cases with detection and recovery: focusedFileIndex out of bounds (clamp + warn log), null scroll target ref (skip scroll, next nav recalculates), null sidebar scrollbox ref (skip sidebar scroll), terminal resize mid-navigation (recalculate at new viewport), whitespace toggle reducing file count (clamp index), navigation during screen transition (keybindings suppressed). 3 failure modes documented: silent scroll failure (state still consistent), sidebar desync (next navigation re-syncs), performance under 500 files (<16ms target, pre-compute cumulative offsets if needed).
