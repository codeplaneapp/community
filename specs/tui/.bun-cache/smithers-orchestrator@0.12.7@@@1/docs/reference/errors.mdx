## Overview

Smithers uses structured errors with machine-readable codes. Every error thrown by the framework is an instance of `SmithersError`:

```ts
import { SmithersErrorInstance, isSmithersError } from "smithers-orchestrator";
import type { SmithersErrorCode } from "smithers-orchestrator";
```

| Export | Kind | Description |
|---|---|---|
| `SmithersErrorInstance` | class | The `SmithersError` class (renamed to avoid conflict with the type alias) |
| `isSmithersError(err)` | function | Type guard ‚Äî returns `true` if the value has a `code` property |
| `errorToJson(err)` | function | Serializes any error (including `SmithersError`) to a JSON-safe object |
| `SmithersErrorCode` | type | Union of all known error codes with `(string & {})` escape hatch |

## Using Error Codes

```ts
import { isSmithersError } from "smithers-orchestrator";
import type { SmithersErrorCode } from "smithers-orchestrator";

try {
  await runWorkflow(workflow, { input: {} });
} catch (err) {
  if (isSmithersError(err)) {
    switch (err.code) {
      case "INVALID_INPUT":
        console.error("Bad input:", err.message);
        break;
      case "AGENT_CLI_ERROR":
        console.error("Agent failed:", err.message);
        break;
      default:
        console.error(`[${err.code}] ${err.message}`);
    }
  }
}
```

## Engine Errors

Thrown during workflow run lifecycle.

| Code | When | Details |
|---|---|---|
| `INVALID_INPUT` | Run input fails validation or has wrong shape | ‚Äî |
| `MISSING_INPUT` | Resume attempted without existing input row | ‚Äî |
| `MISSING_INPUT_TABLE` | Input table not found in database | ‚Äî |
| `RESUME_METADATA_MISMATCH` | Resume run ID or metadata doesn't match stored run | ‚Äî |
| `UNKNOWN_OUTPUT_SCHEMA` | Output table referenced by a task is not in the schema registry | ‚Äî |
| `INVALID_OUTPUT` | Agent response is not valid JSON or cannot be parsed | ‚Äî |
| `WORKTREE_CREATE_FAILED` | Git or jj worktree creation failed | `{ worktreePath, vcsType, branch? }` |
| `VCS_NOT_FOUND` | No git or jj repository found from the root directory | `{ rootDir }` |

## Component Errors

Thrown during DOM extraction or component rendering.

| Code | When | Details |
|---|---|---|
| `TASK_ID_REQUIRED` | A `<Task>` is missing its `id` prop or `id` is not a string | ‚Äî |
| `TASK_MISSING_OUTPUT` | A `<Task>` is missing its `output` prop | `{ nodeId }` |
| `DUPLICATE_ID` | Two tasks, loops, or worktrees share the same ID | `{ kind: "task"\|"ralph"\|"worktree", id }` |
| `NESTED_LOOP` | A `<Loop>` (Ralph) is nested inside another `<Loop>` | ‚Äî |
| `WORKTREE_EMPTY_PATH` | A `<Worktree>` has an empty or missing `path` prop | ‚Äî |
| `MDX_PRELOAD_INACTIVE` | Task prompt resolved to `[object Object]` ‚Äî MDX preload is not active | ‚Äî |
| `CONTEXT_OUTSIDE_WORKFLOW` | `useCtx()` called outside a `<Workflow>` created by `createSmithers()` | ‚Äî |
| `MISSING_OUTPUT` | `ctx.output()` called for a node that hasn't produced output yet | `{ nodeId, iteration }` |

## Tool Errors

Thrown by the built-in tool implementations (read, write, edit, bash, grep).

| Code | When | Details |
|---|---|---|
| `TOOL_PATH_INVALID` | Path argument is not a string | ‚Äî |
| `TOOL_PATH_ESCAPE` | Resolved path escapes the sandbox root directory (including via symlink) | ‚Äî |
| `TOOL_FILE_TOO_LARGE` | File exceeds `maxOutputBytes` limit | ‚Äî |
| `TOOL_CONTENT_TOO_LARGE` | Write content exceeds `maxOutputBytes` limit | ‚Äî |
| `TOOL_PATCH_TOO_LARGE` | Edit patch exceeds `maxOutputBytes` limit | ‚Äî |
| `TOOL_PATCH_FAILED` | Unified diff patch could not be applied | ‚Äî |
| `TOOL_NETWORK_DISABLED` | Bash command attempts network access when `allowNetwork` is false | ‚Äî |
| `TOOL_GIT_REMOTE_DISABLED` | Bash command attempts git remote operations when `allowNetwork` is false | ‚Äî |
| `TOOL_COMMAND_FAILED` | Bash command exited with a non-zero exit code | ‚Äî |
| `TOOL_GREP_FAILED` | `rg` (ripgrep) exited with error code 2 | ‚Äî |

## HTTP API Errors

These are server-internal `HttpError` codes returned in the JSON error response body. They are **not** `SmithersError` instances.

| Code | Status | When |
|---|---|---|
| `INVALID_REQUEST` | 400 | Request body or query parameters are invalid |
| `PAYLOAD_TOO_LARGE` | 413 | Request body exceeds `maxBodyBytes` |
| `INVALID_JSON` | 400 | Request body is not valid JSON |
| `SERVER_ERROR` | 500 | Unexpected server error |
| `UNAUTHORIZED` | 401 | Missing or invalid authorization token |
| `WORKFLOW_PATH_OUTSIDE_ROOT` | 400 | Workflow path resolves outside the server root directory |
| `RUN_ID_REQUIRED` | 400 | `runId` is required when `resume` is true |
| `RUN_ALREADY_EXISTS` | 409 | A run with the given ID already exists |
| `RUN_NOT_FOUND` | 404 | No run found with the given ID |
| `RUN_NOT_ACTIVE` | 409 | Run is not currently active (cannot cancel) |
| `NOT_FOUND` | 404 | Route or resource not found |
| `DB_NOT_CONFIGURED` | 400 | Server database not configured (required for listing runs) |
