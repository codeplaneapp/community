# TUI_DIFF_FILE_TREE

Specification for TUI_DIFF_FILE_TREE.

## High-Level User POV

The diff file tree is the sidebar panel on the left side of the diff viewer screen. It provides an at-a-glance inventory of every file changed in a diff — whether from a single jj change or a landing request — with keyboard-driven navigation to jump between files. Each entry shows a colored change type icon (A/D/M/R/C), the file path (truncated from left with …/ when needed), and a compact stat summary (+N -M). The sidebar is 25% width at standard terminal size (120×40+), hidden by default at minimum size (80×24), and toggleable with Ctrl+B. It includes an inline search filter activated by / for quick file lookup in large diffs, a summary line showing total file count and aggregate stats, and synchronized cursor tracking with the main diff content pane via ]/[ navigation.

## Acceptance Criteria

The file tree renders as the left sidebar of the diff viewer populated from FileDiffItem[] via useChangeDiff() or useLandingDiff() hooks. Each entry displays change type icon (A green/D red/M yellow/R cyan/C cyan), file path, and stat summary (+N -M with green/red coloring). Focused entry uses reverse-video. j/k/Up/Down navigates entries; Enter jumps to file and transfers focus to content; G goes to bottom; gg to top; Ctrl+D/U pages. / activates case-insensitive search filter (128 char max) with incremental narrowing; Esc clears; Enter selects first match. Ctrl+B toggles sidebar; Tab/Shift+Tab switches focus zones. Tree cursor follows ]/[ content navigation. Summary line shows "N files +X -Y" (filtered: "N of M files"). Status bar shows "File N of M" always and context-specific hints. Paths truncated from left with …/; renames show old → new; binary files suffixed [bin]; permission-only [mode]. Responsive: hidden <120 cols, 25% width ≥120, extra padding ≥200. Resize auto-hides/shows respecting manual toggle state and preserves focus/filter/scroll. 500-file cap with truncation indicator. Edge cases: empty diff shows "(No files changed)", single file makes j/k no-ops, rapid keys processed sequentially.

## Design

Two-pane layout: file tree sidebar (left, conditionally rendered) + main diff content (right, flexGrow=1). Sidebar uses <box> with borderRight="single" containing: summary line (1 row), optional search input, separator, <scrollbox> with file entries. Each entry is a 1-row <box> with inverse styling when focused, containing change type icon <text>, file path <text flexGrow=1>, optional [bin]/[mode] suffix, and stat <text>. Focus model has two zones (tree/content) toggled by Tab/Shift+Tab. State: focusedFileIndex, focusZone, sidebarVisible, sidebarManuallyToggled, searchActive, searchQuery, scrollOffset. Keybindings: j/k/G/gg/Ctrl+D/Ctrl+U for tree nav, Enter to select, / for search, Esc to clear, Ctrl+B to toggle sidebar, ]/[ for file nav (synced). Data from useChangeDiff()/useLandingDiff() hooks. Responsive breakpoints: 80-119 (hidden, 30% when toggled, min 24 cols), 120-199 (visible, 25%), 200+ (visible, 25% + padding). Change type icons: A=green(34), D=red(196), M=yellow(178), R=cyan(37), C=cyan(37). Status bar hints vary by focus zone. Resize transitions respect sidebarManuallyToggled flag.

## Permissions & Security

TUI requires authentication at bootstrap — unauthenticated sessions never reach the diff screen. File tree is read-only, derived entirely from the diff API response with no additional API calls or write operations. Public repos viewable by any authenticated user; private repos return 404 to users without read access (no existence leak). Token loaded from CLI keychain or CODEPLANE_TOKEN env var, passed as Bearer token, never displayed/logged. 401 responses propagate to auth error screen. No additional rate limiting for tree operations (all client-side); rate limiting handled at diff fetch level. Search input treated as literal substring (no regex/shell evaluation), length-limited to 128 chars. File paths from API responses, not user input. All rendering through OpenTUI's text component for ANSI escaping.

## Telemetry & Product Analytics

Key events: tui.diff.file_tree.viewed (diff opened with sidebar visible — source, repo, file_count, terminal dims, breakpoint), tui.diff.file_tree.navigate (cursor moved — direction, from/to index, total), tui.diff.file_tree.select_file (Enter on entry — file details, method enter|search_enter), tui.diff.file_tree.file_synced (cursor follows ]/[ — direction), tui.diff.file_tree.search_opened (/ pressed — file_count), tui.diff.file_tree.search_completed (search resolved — query_length, match_count, outcome selected|cleared|esc), tui.diff.file_tree.sidebar_toggled (Ctrl+B — new_state, trigger manual), tui.diff.file_tree.sidebar_auto_hidden (resize — old/new width), tui.diff.file_tree.focus_changed (zone switch — new_focus, method tab|shift_tab|enter). Common props: session_id, timestamp, terminal_width, terminal_height, viewer_id. Targets: >40% interaction rate, >25% selection rate, >10% search usage for ≥5-file diffs, >30% sidebar toggle at 80-col, 100% sync accuracy, <50ms render, <16ms filter latency, 60 entries/sec navigation throughput.

## Observability

Logging (stderr, level via CODEPLANE_LOG_LEVEL default warn): info for tree rendered (repo, source, file_count, sidebar_visible, terminal_width), file selected (path, index, change_type), search filter used (query_length, match_count, outcome); debug for cursor moved, tree cursor synced, sidebar toggled, search filter applied (with filter_time_ms), resize layout recalc, scroll position updated. Error cases: empty file_diffs shows "(No files changed)", >500 files renders first 500 with truncation indicator + log warning, resize hides sidebar and transfers focus, resize preserves search filter, sidebar < 24 cols force-hides, missing path field skips entry + warns, unknown change_type shows ? icon in muted + warns, rapid j/k batches visual updates, Shift+Tab no-op when sidebar hidden, ]/[ clears search filter before navigating, React error boundary isolates tree errors from content pane. Failure modes: diff fetch failure leaves tree empty with error in content (R retries), partial data skips malformed entries, scrollbox virtualizes for memory, toggle during navigation is synchronous, special regex chars treated as literals.
