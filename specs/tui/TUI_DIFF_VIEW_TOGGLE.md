# TUI_DIFF_VIEW_TOGGLE

Specification for TUI_DIFF_VIEW_TOGGLE.

## High-Level User POV

The diff view toggle is the mechanism by which a developer switches between unified and split (side-by-side) diff layouts while reviewing code in the Codeplane TUI's diff screen. It is activated by pressing the `t` key anywhere on the diff screen. The toggle is instantaneous — no loading or API re-fetch — operating entirely on already-fetched diff data. Unified mode (default) shows additions/deletions interleaved in a single column. Split mode shows old file content on the left and new on the right with synchronized scrolling. The current mode is reflected in the status bar as `[unified]` or `[split]`. Split mode requires ≥120 column terminal width; pressing `t` below that threshold flashes a rejection message for 3 seconds. If the terminal resizes below 120 columns during split mode, the view auto-reverts to unified with a flash notification. The user's preferred mode is remembered so a single `t` press restores split after a forced revert. The toggle applies globally to all files, persists within a diff session, and resets to unified on new diff screens. Rapid keypresses are debounced at 100ms.

## Acceptance Criteria

Core toggle: `t` toggles unified↔split; `<diff>` view prop set accordingly; syncScroll=true in split; no API re-fetch; no loading indicator; global across all files. Width constraint: split requires ≥120 columns (inclusive); rejected toggle flashes 'Split view requires 120+ column terminal' for 3s; flash replaces status bar temporarily. Auto-revert: resize <120 during split reverts to unified synchronously with flash message; resize back above 120 does NOT auto-restore split; user's preferred mode remembered for next `t` press. Scroll preservation: same logical line at viewport top before/after toggle in both manual and automatic switches. Status bar: shows [unified] or [split]; keybinding hint always reads `t:view`; updates synchronously. Split layout: with sidebar 37.5%/37.5%/25%; without sidebar 50%/50%; each pane has own line number gutter; added lines right-only green, deleted left-only red, context both sides aligned. Debounce: 100ms window, input-layer. Lifecycle: default unified, persists across file nav/hunk ops/whitespace toggle, resets on new screen, not persisted to config. Keybinding context: works in main content, file tree, hunk focus; blocked by help overlay, comment form, command palette. Edge cases: no-op during loading/error/empty states; works on single-file and binary diffs; collapsed hunks preserved; 16-color fallback uses sign differentiation.

## Design

No new screen-level components. State: `viewMode` ('unified'|'split', default 'unified'), `preferredMode` (tracks user's explicit choice for post-revert restoration), `flashMessage` (string|null, auto-clears after 3s), `lastToggleTimestamp` (ref, debounce tracking). Uses `useTerminalDimensions()` for width check (canSplit = width ≥ 120), `useOnResize()` for auto-revert, `useKeyboard()` for `t` binding with overlay guard. `<diff>` component receives `view={viewMode}`, `syncScroll={viewMode === 'split'}`, plus syntax highlighting and color props. Layout: unified = sidebar(25%) + single column(75%); split = sidebar(25%) + left pane(37.5%) + right pane(37.5%); sidebar hidden = 50%/50%. Status bar center section renders mode indicator `[unified]`/`[split]`. Flash messages take over full status bar width for 3s duration. At 80×24 minimum, split unavailable, `t` always rejected with flash. Line number gutters: 5-char at standard (120×40), 6-char at large (200×60+). Toggle debounced at input layer with Date.now() ref comparison.

## Permissions & Security

The diff view toggle is purely client-side — it changes how already-fetched diff data is rendered. No API calls are made and no specific authorization role is required beyond the repository read access inherited from the diff screen. No token is sent or validated during toggle. No server-side rate limiting applies. Input is limited to the `t` keypress; the `view` prop is always one of two string literals and is never interpolated from user input.

## Telemetry & Product Analytics

Events: `tui.diff.view_toggled` (from_mode, to_mode, terminal dimensions, sidebar_visible, file_count, trigger='keypress'), `tui.diff.view_toggle_rejected` (terminal dimensions, attempted_mode='split'), `tui.diff.view_auto_reverted` (terminal dimensions, previous_width, from/to modes, trigger='resize'). Common properties: session_id, timestamp, user_id, diff_source ('change'|'landing'), repo. Success indicators: >20% diff session adoption, track split preference ratio, <15% rejection rate at narrow terminals, <5% auto-revert frequency, positive correlation between toggle usage and session duration.

## Observability

Logging: info `diff.view.toggled` with from/to mode, terminal_width, trigger; warn `diff.split.unavailable` with dimensions when rejected; warn `diff.auto_switch_unified` with current/previous width on resize revert; debug `diff.toggle.debounced` with elapsed_ms; debug `diff.scroll.preserved` with line_index and modes. Failure modes: OpenTUI version mismatch (view prop unsupported → falls back to unified-only with warn log; syncScroll ignored → cosmetic degradation), useOnResize unavailable (no auto-revert, user manually toggles), inaccurate terminal dimensions (threshold check may be wrong), 16-color terminal (background colors degrade but +/- signs remain). Error recovery: diff component throw caught by error boundary with R-to-retry; flash timer cleanup on unmount via useEffect return; preferredMode/viewMode desync self-heals on next toggle.
